# 관악구 공시가격 형평성 진단 플랫폼

> 공간 통계 기반 공시가격 형평성 진단 · 서울특별시 관악구 공동주택 54,132세대

---

## 프로젝트 개요

공동주택 공시가격은 재산세, 건강보험료, 기초연금 등 60여 개 행정 지표의 기준이 되는 핵심 공공데이터입니다. 그러나 일반 시민은 자신의 주택 공시가격이 주변 대비 적정한지 알 방법이 없습니다.

본 프로젝트는 공간자기회귀모형(SAR) 으로 관악구 공동주택의 적정 실거래가를 추정하고, 국토교통부 업무요령 기반 가격 구간별 현실화율을 계층적으로 적용하여 세대별 적정 공시가격을 도출합니다. 실제 공시가격과의 비율로 형평성을 진단하고 웹 플랫폼으로 제공합니다.

## 실행 순서

### Step 1a — 지오코딩 및 거리 변수 계산

```bash
python3 step1_geocoding_distance.py
```

- 카카오 REST API로 단지별 위경도 수집 (3,327/3,328건, 99.97%)
- Haversine 공식으로 업무지구·지하철역·근린공원까지 최단거리 계산
- 출력: `data/Int_Data_geo.csv`

**거리 계산 (Haversine):**
```
dist = 2R × arcsin(√(sin²(Δlat/2) + cos(lat₁)·cos(lat₂)·sin²(Δlon/2)))
```

| 기준점 | 내용 |
|--------|------|
| 업무지구 | 여의도·강남·광화문 3곳 중 최단거리 |
| 지하철역 | 관악구 인근 16개 역 (2호선·4호선·7호선·신림선) |
| 근린공원 | 관악구 내 10개 공원 |

### Step 1c — 전세 실거래가 수집 및 126% 진단

```bash
python3 step1c_jeonse.py
```

- 국토교통부 전월세 실거래가 API (`apis.data.go.kr`) 호출
- 수집 기간: 최근 5개년 60개월 (오늘 기준 자동 계산)
- 수집 유형: 아파트 + 오피스텔 + 연립다세대
- 전세 필터: `보증금 > 0 AND 월세 = 0`
- 대표값: 단지별·면적별 **가장 최근 거래가** 채택

**수집 결과 (2021.04 ~ 2026.03):**

| 유형 | 건수 |
|------|------|
| 아파트 | 17,734건 |
| 오피스텔 | 3,229건 |
| 연립다세대 | 13,425건 |
| **합계** | **34,388건** |

**매칭 전략:**
- 1차: 단지명 + 전용면적(반올림) → 48,211건 (89.1%)
- 2차: 도로명 키워드 + 면적 → +59건
- 최종: **48,270건 (89.2%)**

**126% 초과 판단:**
```
전세대출 불가 조건: 전세보증금(만원) > 공시가격(원) ÷ 10,000 × 1.26
```

| 구분 | 건수 | 비율 |
|------|------|------|
| 전세대출 불가 (126% 초과) | 8,418건 | 15.6% |
| 전세대출 가능 (126% 이하) | 39,852건 | 73.6% |
| 전세 데이터 없음 (미확인) | 5,862건 | 10.8% |

### Step 2 — 공간적 자기상관 진단

```bash
python3 step2_spatial_autocorrelation.py
```

**공간 가중치 행렬:** KNN(k=8), Row Standardization

**전역 Moran's I:**
```
I = (n / S₀) × [Σᵢ Σⱼ wᵢⱼ(xᵢ - x̄)(xⱼ - x̄)] / [Σᵢ(xᵢ - x̄)²]
```

| 변수 | Moran's I | p-value | 판정 |
|------|-----------|---------|------|
| 공시가격 | 0.8906 | 0.001 | 유의 |
| 실거래가 | 0.9105 | 0.001 | 유의 |
| **평가비율** | **0.4069** | **0.001** | **유의 → SAR 채택** |

**LISA 클러스터:**

| 유형 | 의미 | 세대 수 |
|------|------|---------|
| HH(과대집중) | 고평가 지역 내 고평가 주택 | 4,694건 |
| LL(과소집중) | 저평가 지역 내 저평가 주택 | 4,724건 |
| HL(주변과소) | 주변보다 높게 평가 | 1,406건 |
| LH(주변과대) | 주변보다 낮게 평가 | 1,196건 |
| 비유의 | 유의한 군집 없음 | 42,112건 |

- 출력: `data/Int_Data_geo.csv` (`lisa_cluster` 컬럼 추가)

### Step 3 — SAR 모형 추정 및 형평성 진단

```bash
python3 step3_model.py
```

#### 종속변수

`log(median_price)` — 매매 실거래가 그룹 중앙값의 로그 변환값 (단위: 만원)

#### 독립변수 (9개)

| 변수명 | 설명 | 수집 출처 |
|--------|------|-----------|
| log_land_price | log(공시지가, 원/㎡) | 공공데이터포털 CSV |
| age | 건축경과연수 (2026 - 건축연도) | 원본 데이터 |
| log_priv_area | log(전용면적, ㎡) | 원본 데이터 |
| floor_clean | 층수 (지하층 → 0 클리핑) | 원본 데이터 |
| dist_cbd | 업무지구까지 최단거리 (km) | Step 1a |
| dist_subway | 지하철역까지 최단거리 (km) | Step 1a |
| dist_park | 근린공원까지 최단거리 (km) | Step 1a |
| block_name_봉천동 | 봉천동 더미 (남현동 기준) | 원본 데이터 |
| block_name_신림동 | 신림동 더미 (남현동 기준) | 원본 데이터 |

#### 비교모형: Hedonic OLS

```
log(실거래가) = β₀ + β₁·log(공시지가) + β₂·건축경과연수
              + β₃·log(전용면적) + β₄·층수
              + β₅·업무지구거리 + β₆·지하철역거리 + β₇·근린공원거리
              + β₈·봉천동더미 + β₉·신림동더미 + ε
```

R² = 0.7889, RMSE(log) = 0.2736, MAE = 11,703만원
잔차 Moran's I = 0.701 (p<0.001) → SAR 필요성 재확인

#### 주 모형: SAR (공간자기회귀모형)

```
log(실거래가) = β₀ + Σ βₖ·Xₖ + ρ·W·log(실거래가) + ε
```

`W·log(실거래가)`: 인접 8개 이웃 세대의 log 실거래가 가중 평균

**추정 결과 (White 표준오차):**

| 변수 | 계수 | z값 | p값 |
|------|------|-----|-----|
| 상수 | 0.533 | 4.59 | <0.001 |
| log(공시지가) | 0.229 | 19.02 | <0.001 |
| 건축경과연수 | -0.008 | -57.32 | <0.001 |
| log(전용면적) | 0.696 | 58.92 | <0.001 |
| 층수 | 0.008 | 47.78 | <0.001 |
| 업무지구거리 | -0.002 | -1.21 | 0.228 |
| 지하철역거리 | 0.026 | 6.50 | <0.001 |
| 근린공원거리 | -0.031 | -12.04 | <0.001 |
| 봉천동 더미 | 0.006 | 0.98 | 0.327 |
| 신림동 더미 | -0.040 | -7.74 | <0.001 |
| **W·log(실거래가) ρ** | **0.378** | **28.65** | **<0.001** |

**모형 성능 비교:**

| 지표 | OLS | SAR | 개선 |
|------|-----|-----|------|
| R² | 0.7889 | 0.9040 | +11.5%p |
| RMSE(log) | 0.2736 | 0.1861 | **-32.0%** |
| MAE(만원) | 11,703 | 7,450 | -36.3% |

#### 적정 공시가격 역산

```
현실화율 = 실제 공시가격(원) ÷ 실거래가 중앙값(원)
적정 공시가격 = SAR 추정 실거래가(만원) × 최종 현실화율 × 10,000
```

**현실화율 계층적 적용 (국토부 2026 업무요령 기준):**

| 단계 | 기준 | 적용 조건 |
|------|------|-----------|
| 1단계 | 가격 구간별 현실화율 | 항상 기본 적용 |
| 2단계 | 그룹(건물×층구간) 중앙 현실화율 | 그룹 내 5세대 이상 시 우선 |
| 대체 | 전체 중앙값 59.1% | 결측 시 대체 |

| 가격 구간 | 현실화율 |
|-----------|---------|
| 3억 미만 | 60.0% |
| 3억 ~ 6억 | 59.4% |
| 6억 ~ 9억 | 58.4% |
| 9억 초과 | 59.5% |

- 그룹 현실화율 적용: 45,516건 (84.1%)
- 구간 현실화율 적용: 8,616건 (15.9%)

#### 형평성 등급 — IAAO 국제감정평가사협회 기준

```
형평성 비율 = 실제 공시가격 ÷ 적정 공시가격
```

| 등급 | 기준 | 세대 수 | 비율 |
|------|------|---------|------|
| 심각과대 | > 1.25 | 3,515건 | 6.5% |
| 과대 | 1.10 ~ 1.25 | 10,368건 | 19.2% |
| **적정** | **0.90 ~ 1.10** | **27,898건** | **51.5%** |
| 과소 | 0.75 ~ 0.90 | 7,961건 | 14.7% |
| 심각과소 | < 0.75 | 4,390건 | 8.1% |

- 출력: `data/Int_Data_result.csv` (54,132행)

## 방법론적 한계

- **향별·위치별 효용비 미반영**: 공공데이터로 세대별 향(남향/북향 등) 수집이 불가하여 제외. 민간 플랫폼 데이터 연계로 향후 보완 가능
- **현실화율 순환 논리**: 공시가격 자체를 현실화율 계산에 사용하므로 공시가격 편향 시 현실화율도 편향될 수 있음
- **연구 목적**: 완벽한 적정 공시가격 산출이 아닌, 공시가격 형평성의 공간적 패턴 진단 및 정보 비대칭 해소
